<JustAMiddlewareLink /> This means that, in principle, you can use any [AWS](https://aws.amazon.com) deployment tool.

## Full-stack

Tools for deploying full-stack apps (frontend + backend + database) to AWS:
 - [Flightcontrol](https://www.flightcontrol.dev?ref=vike)
 - [Argonaut](https://www.argonaut.dev)

## AWS Lambda

Tools for deploying to [AWS Lambda](https://aws.amazon.com/lambda/):
- [Claudia.js](https://claudiajs.com/)
- [Apex Up](https://github.com/apex/up)
- [The Serverless Framework](https://www.serverless.com/)
- [SAM](https://aws.amazon.com/serverless/sam/)

Examples of using Vike and AWS Lambda with The Serverless Framework:
 - [GitHub > Strydom/vite-production-serverless-build](https://github.com/Strydom/vite-production-serverless-build)
   - See the used [workarounds](https://github.com/vitejs/vite/discussions/13212#discussioncomment-5913756)
 - [GitHub > jamesladd/aws-vite-ssr](https://github.com/jamesladd/aws-vite-ssr)

> vite-plugin-ssr was the [previous name of Vike](https://vite-plugin-ssr.com/vike).
